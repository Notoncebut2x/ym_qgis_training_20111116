What are Maps?

As you can see from Johannes Vermeer's ~1668 painting titled The Geographer, they used to be painstaking hand drawn documents. Lucky for you and me, since the 1980s maps have mostly made with the help of computers.
Maps made on computers are built from different datasets. This data can be a single point, a collection of polygons, or a photo like recording of color and temperatures. The map on this slides consists of point vector data for the location of cities, line vector data for the states, and a raster dataset to represent the percentage of days the temperature exceeded 0 or 100 degrees farenheit.
With data we calculate numbers and make pictures (like a maps!). When calculations are preformed on data we analyze it. This means we transform the data by doing things like aggregating, summarzing it. The process of making a picture from data is called symbolization this is when you make decisions on what visual element will represent different parts of the dataset.
Symbolization and analysis are the two parts needed to make a map. This training will introduce you to these basic concepts and then have you apply different techniques for symbolization and analysis. But before we do that... why should we even care about maps in the first place?

GIS Basics

Now that we have some basic geographic techniques down, lets get started with some basic GIS concepts.

Geographic phenomena on our map can be discrete or continuous.

The geographic phenomena we want to map can be discrete or continuous.

Continuous types changes gradually, not abruptly ex: temperature, elevation, salinity.

Continous geographic phenomena changes gradually and is not abrupt. This can be for things like temperature, elevation, water salinity.

Discrete types are divided into mutually exclusive parts, with all locations in one part having the same value.

Other types of geographic phenomena are discrete. This means that they can be divided into distinct parts with everything in the value having the exact same value. For example country boundaries are discrete as are a building's footprint, or a road.

Different types of data can be used to represent our geographic phenomenon.

To store and symbolize our geographic phenomenon there are different types of data.

But you have to make sure that the right data is used to represent the right type of phenomenon!!!

Matching our data to the right type of phenomenon is crucial.

  • Nominal Data provides an identifier to discriminate between values.
  • Ordinal Data can be put in some natural sequence.
  • Interval Data allows for simple forms of computation like + and -.
  • Ratio Data allows for most, if not all, arithmetic computations.

In general, our data can take four forms: Nominal which provides an identifier to distinguish different values...an example would be the names of countries to dingish the differen boundaries from one another. Ordinal data which can be put into a natural sequence think data that would have high, medium, low associated with it. Interval data which allows for simple forms of computation like addition and subtraction. Examples of interval data would be: time, date, temperature, and ph scale. Finally, ration data which allows for most, if not all, arthmetic computations. Think of different data that has a number associated with it like population statics broken down by gender or age.

GIS File Types

Before I pass the presentation over to Christine, I will talk about how this data is stored.

Map data is stored as a Vector or a Raster.

All of the different types of data we just discussed can be stored as a vector or raster file type. However there is usally one format that is more efficant than the other.

A raster is a set of regularly spaced cells with associated values.

The value of the cell is assumed to be valid for all the area within it.

A raster is a grid...regularly spaced cells with a value associated with it. The value of the cell is assumed to be valid for all of the area within it.
In this example you can see the black part of the grid that forms the face is given a value of 1 and the white cells a value of 0.
This is a more complex example where you can see the state of Oregan in the USA and the pixels that make up this raster relate to elevation.

Vectors, unlike rasters, attempt to explicitly georeference the geographic phenomena.

The other type of way data is stored for GIS software are vectors. Unlike rasters, vectors attempt to explicitly georeference the geographic phenomena they represent.

They can be a single point (x, y)or(x, y, z).

Vectors can be a single point (x,y) or with a height (x,y,z)

They can be a line. Which requires two or more nodes and a line drawn between them.

Vectors can also be a line which requires two or more ordered points (also called nodes) with a line drawn between them

Another type of vectors are polygons. They are a collection of nodes and arcs.

Finally, vectors can be polygons. These are a colletion of nodes with arcs connecting them. The first and last node are the same point meaning that the it forms a closed shape.

Each section of vector data can have thematic data associated with it.

Each part of a vector dataset can have different thematic (also called attribute) data associated with it. It can include nominal data like the name of the feature and ratio data like the area of the feature. And any type of data associated with it.

Exporting Data from OSM

Over the years there has been a lot of different tools and websites developed for extracting data from OSM. Today I will show you three of my favorite.

  • HOT OSM Export Tool
  • Overpass Turbo
  • Geofabrik

HOT Export Tool

  • Pros
    • Easy to use
    • Exports data in a variety of formats
    • OSM data is catorigized into easy to understand categories
  • Cons
    • Can be slow when the OSM servers are busy or a lot of exports have been requested

Overpass Turbo

  • Pros
    • Requires knowledge of OSM data structure
    • You an view it before you download it!
    • Can be incorporated into programming scripts
    • Works very quickly for small datasets
    • Has a wizard to assist with Queries
  • Cons
    • Requires learning a progamming language
    • Requires knowledge of OSM data structure

GeoFabrik

  • Pros
    • Provides standard ready to use GIS Files (.shps!)
  • Cons
    • "Fremium" model meaning data may be incomplete
    • The data can be a little outdated
